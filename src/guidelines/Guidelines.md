# System Guidelines

Use this file to provide the AI with rules and guidelines you want it to follow.
Keep responses consistent with our design system, ensuring usability, clarity, and brand alignment.

## General Guidelines
* 僅在必要時使用絕對定位，優先使用 Flexbox / Grid，確保頁面保有響應式設計。
* 需維持程式碼整潔，helper functions 與 components 應獨立存放。
* 保持檔案輕量化，避免冗餘結構。
* 一致性：顏色、字體、間距、狀態必須來自 Design Tokens 和 library，並使用既有的 components。
* 需使用 Design Tokens（顏色/字體/間距/圓角）。
* 需確定每個狀態需有明確的視覺/文字回饋
* 保持流程簡潔，符合使用者體驗
* 若使用者有提供版型操考圖片，需根據提供的圖片作為版面設計呈現參考，但仍須優先使用既有的 components。

## Design Tokens
### Color Palette
* Neutral：Black、Grey 系列 (100–900)、White
* Primary：主色為 #01579B，並有 100–500 層級
* Secondary：主色為 #568B53，並有 100–500 層級
* Warn：高風險色 #C00000 搭配 100–500 層級
* Theme Colors：Light Green、Teal、Cyan、Blue、Orange、Red

## Scale & Spacing
* Font 內文文字標準為14px

## Patterns
### 操作行為：通常搭配 button 進行操作
* 新增/編輯/檢視/刪除/複製/取消/關閉/下一步：依重要程度決定使用圓形 Button 或 CwButton
* 完成或錯誤時需有明確 Toast 視覺回饋
* 重大操作需透過 popup 搭配或 warning 警告使用者

### 全域框架
* Header：顯示系統名稱、登入者資訊、通知、選單收合按鈕。寬度為 100%，可依據使用者螢幕寬度自動縮放，確保在各種裝置上皆能完整顯示。
* SideMenu：支援折疊、搭配多層結構，最高層結構必須搭配icon以利縮合時辨識。展開時寬度固定為 250px，高度為 100%，隨視窗高度自動延伸。
* Footer：顯示版本與維護單位。
* 右側通知欄：使用 drawer 並支援折疊，展開時寬度固定為 400px，高度為 100%，隨視窗高度自動延伸。
* 內容區域：寬高依視窗尺寸自動調整，左右邊距固定為 30px，上下邊距為 20px，以確保內容具有良好的可讀性與一致的視覺留白。

### 狀態
* Empty State：基本須包含「圖像+說明文字」，若有下一步動作的引導，可加入CwButton作為動作提示。
* Disabled：呈現灰階，並不可點擊。又細分為預設禁用/唯讀/隱藏，禁用時滑鼠旁會帶有禁用的icon作為提示。
* Loading：全螢幕遮罩並在中間顯示運轉圖像，Loading 時禁止其他操作。
* Status Indicators：紅=錯誤、橙=警告、綠=成功、灰藍=中立。
* 錯誤訊息提示：當頁面為表單，送出時需檢查必填及資料正確性，檢查錯誤時需於未填寫或填錯的元件下方呈現紅色的錯誤提示文字。

### UI Patterns
* 資訊揭露：使用 Tooltip / Popup / Drawer，不可多層嵌套，根據資訊的多寡選擇元件。
* 不應將必填欄位或關鍵任務操作藏在收合區，整體操作需符合使用者體驗
* 流體版面：以桌面顯示為主，RWD 為輔，根據版面決定元件的使用。
* **版面區域劃分（重要）**：
  - 同一個群組的範圍不需要加入邊框線（border）和 padding
  - 優先使用上下 margin（如 space-y-6）來製造不同區域的分離感
  - 必要時可以加入淺灰分隔線 `<hr className="border-t border-border/30" />` 來區分大區塊
  - 禁止使用 `bg-card p-6 rounded-[var(--radius-card)] border border-border` 這類樣式包裹內容區域
  - 除非有特別說明，否則一律遵循此原則
* **按鈕位置規則（重要）**：
  - 當頁面的內容有確認、儲存、提交等主要操作按鈕時，必須放置在右下方
  - 使用 `justify-end` 讓按鈕靠右對齊
  - 按鈕順序依然遵循：取消(outline) > outline > 實心 > 刪除(實心)
  - 適用於表單編輯頁面、資料詳細頁面等需要儲存或確認操作的場景
* 通知分類：Info：提供補充說明，與任務無關的附加資訊 / Success：任務已順利完成 / Warning：行為可能產生風險或意外結果 / Error：操作失敗，需立即處理或停止行為 / Question：系統無法辨別意圖，需重新操作

### 層級
* 根據重要程度來控制各元件層級的優先順序
* z-index：Drawer(1000) 作為側邊操作，Toast(2000) 與 Tooltip(3000) 提供輕量提示，Dropdown/Select(3200) 確保不被 Drawer 遮蓋，Popup(4000) 負責高優先度操作。若右下角設有快速點擊icon(5000) 永遠固定可見，快速點擊icon所展開的iframe(6000) 為最高層主要互動介面。
* 提示型元件不應阻斷操作，而關鍵元件則必須具備明確的關閉機制，確保使用流暢與安全。

## Components
### CwHeader
* 頁面頂部元件，固定顯示在所有頁面的上方。
* **尺寸與樣式**：
  - 寬度：100%，可依據螢幕寬度自動縮放
  - 高度：固定 60px
  - 背景色：#7eaa82（綠色）
  - 左右 padding：30px
  - 帶有陰影效果（var(--elevation-sm)）
* **結構組成**：
  - **左側**：選單按鈕（漢堡選單圖標）+ 系統名稱
  - **右側**：通知按鈕（鈴鐺圖標）+ 會員圖標 + 使用者名稱
  - 左右兩側使用 gap-[35px] 和 gap-[14px] 間距
* **互動功能**：
  - 選單按鈕：點擊可切換側邊選單的展開/收合
  - 通知按鈕：點擊開啟通知面板，可顯示紅點提示（hasNotification 屬性）
  - 會員圖標：點擊顯示使用者選單或導航到個人資料
  - 所有按鈕都有 hover 透明度效果
* **使用規則**：
  - 必須固定出現在所有頁面的頂部
  - 提供全域導航和系統資訊
  - 通知紅點用於提示未讀訊息或重要通知
  - 系統名稱和使用者名稱可自訂
* **適用場景**：全域導航、系統識別、使用者資訊顯示、通知提醒。

### CwFooter
* 頁面底部元件，固定顯示在所有頁面的底部。
* **尺寸與樣式**：
  - 寬度：100%
  - 高度：自動適應內容
  - 背景色：白色 (#ffffff)
  - padding：10px
  - 帶有上方陰影效果（0px -2px 15px 0px rgba(196, 201, 211, 0.6)）
* **文字樣式**：
  - 字體：Noto Sans TC，14px，font-weight 400
  - 顏色：黑色 (#1c1c1c)
  - 對齊：居中顯示
  - 單行顯示（whitespace-nowrap）
* **使用規則**：
  - 必須固定出現在所有頁面的底部
  - 顯示版權資訊與維護單位
  - 文字內容應簡潔明確，通常包含年份和公司/組織名稱
  - 避免放置過多資訊或複雜內容
  - 保持單行顯示，確保在不同螢幕寬度下都能完整閱讀
* **主要屬性**：
  - `copyrightText`：版權文字（預設：Copyright © 2025 天下雜誌群）
* **適用場景**：版權聲明、維護單位資訊、法律聲明、技術支援資訊。

### CwSidemenu
* 側邊選單元件，固定顯示在所有頁面的左側，提供主要導航功能。
* **尺寸與樣式**：
  - 展開寬度：250px，收合寬度：64px
  - 高度：100%，隨視窗高度自動延伸
  - 背景色：#568853（綠色）
  - 上下 padding：20px
  - 支援上下 Y 軸捲動（當項目過多時）
* **層級結構**：
  - 支援最多 2 層結構
  - **第一層**：必須有 icon 屬性，方便收合時辨識，可以有 children
  - **第二層**：不需要 icon，不可再有子項目
* **項目樣式**：
  - 項目高度：44px
  - 項目 padding：px-28px py-10px
  - 文字：Noto Sans TC，14px，font-weight 350，白色
  - Icon 與文字間距：10px
  - Icon 尺寸：20px × 20px
* **互動狀態**：
  - **正常狀態**：背景透明
  - **Hover 狀態**：背景色 #7eaa82
  - **選中狀態**：背景色 #96a895
  - **展開/收合圖標**：
    - 收合狀態：向右箭頭（>）
    - 展開狀態：向下箭頭（箭頭旋��� 90 度）
    - 僅在第一層有子項目時顯示
* **功能特性**：
  - 第一層點擊：有子項目時展開/收合，無子項目時觸發導航
  - 第二層點擊：直接觸發導航
  - 收合模式：僅顯示第一層圖標（置中顯示），隱藏文字和子項目
  - 收合模式 Hover 行為：滑鼠 hover 到項目時，會從右側彈出懸浮選單顯示該項目的標題和子項目，可直接點選導航
  - 懸浮選單設計：背景色 #568853，圓角 4px（右側），帶陰影效果，z-index: 5000
  - 支援預設展開的項目列表
  - 追蹤當前選中的項目
* **主要屬性**：
  - `items`：菜單項目列表（必填）
  - `activeId`：當前選中的項目 ID
  - `onItemClick`：項目點擊回調函數
  - `defaultExpandedIds`：預設展開的項目 ID 列表
  - `collapsed`：是否收合（預設 false）
* **使用規則**：
  - 必須固定出現在所有頁面的左側
  - 提供主要導航功能
  - 可配合 CwHeader 的選單按鈕切換展開/收合狀態
  - 第一層項目必須有圖標，第二層項目不需要圖標
  - 層級深度不可超過 2 層
  - 當項目過多超出視窗高度時，可上下捲動查看
  - 收合模式下，icon 必須置中對齊，padding 調整為 px-22px
  - 收合模式下，hover 展開的懸浮選單需有 300ms 的延遲關閉，讓使用者有時間移動滑鼠
* **適用場景**：
  - 全域導航
  - 系統功能分類與組織
  - 中等複雜度的導航結構（2 層以內）
  - 配合 Header 和 Footer 構成完整的頁面框架

### CwTitle
* 頁面標題元件，出現在每一頁內容區域的上方。
* 由標題文字（h2）與 CwBreadcrumbs 元件組成。
* 佈局：標題在左側，麵包屑在右側，使用 justify-between 對齊。
* 標題樣式：使用 Inter 字體，24px，font-weight 500。
* 上下 padding 各 10px。
* 每個頁面都應該使用 CwTitle 元件，提供一致的頁面標題體驗。

### CwBreadcrumbs
* 顯示當前頁面在網站層級結構中的位置。
* 最後一層標示為當前位置，使用深色文字，不可點擊。
* 中間層級的文字使用淺色文字，可點擊，hover 時顯示下劃線，幫助使用者返回上一階層。
* 階層數量最多不超過 5 層，避免影響閱讀與理解。
* 分隔符號使用 "/"，顏色與中間層級相同。
* 通常作為 CwTitle 元件的一部分使用。

### CwButton
* Primary：關鍵操作
* Secondary：支援性操作
* Tertiary：中立性操作
* Warn：高風險刪除/撤回
* Disabled：無效狀態/禁用
* 一個頁面中會讓最主按鈕使用 Primary 實心，其餘都使用 Primary outline。表格內的使用圓形或文字按鈕。
* 當同一個頁面有多個按鈕群組時，根據按鈕的特性加上 icon ，保持層級感與差異性
* **按鈕組排列規則**：當畫面中有多個 CwButton 為一組時，由左至右依序為：取消(outline) > outline > 實心 > 刪除(實心 destructive)
  - 取消按鈕：永遠放在最左邊，使用 Primary outline
  - 次要操作：放在中間偏左，使用 Primary outline
  - 主要操作：放在中間偏右，使用 Primary 實心
  - 刪除按鈕：永遠放在最右邊，使用 Destructive 實心

### CwRoundButton
* 圓形按鈕元件，用於表格內或空間受限的操作按鈕。
* **尺寸與樣式**：固定寬高 24px，圓角 50%，按鈕中心放置圖標。
* **圖標類型**：
  - `view`：檢視圖標（眼睛）
  - `edit`：編輯圖標（鉛筆）
  - `delete`：刪除圖標（垃圾桶）
  - `download`：下載圖標（向下箭頭）
  - `copy`：複製圖標（文件）
  - `more`：更多操作圖標（三個點）
* **顏色變體**：
  - `default`：預設樣式，白色背景，灰色邊框，灰色圖標
  - `primary`：藍色圖標，用於主要操作
  - `destructive`：紅色圖標，用於刪除等危險操作
* **狀態**：
  - 正常：顯示對應顏色的圖標
  - Hover：背景色變化
  - 禁用：半透明顯示，無法點擊
* **重要使用規則**：
  - **必須搭配 CwTooltip 使用**，為使用者提供按鈕功能說明
  - Tooltip 內容應簡潔明確，直接說明操作（如：檢視、編輯、刪除、下載）
  - 禁止單獨使用 CwRoundButton 而不包裹 CwTooltip
* **適用場景**：
  - 表格操作欄位
  - 卡片右上角快捷操作
  - 工具列小型操作按鈕
  - 空間受限需要緊湊佈局的場景
* **範例用法**：
  ```tsx
  <CwTooltip content="檢視">
    <CwRoundButton icon="view" onClick={handleView} />
  </CwTooltip>
  
  <CwTooltip content="刪除">
    <CwRoundButton icon="delete" variant="destructive" onClick={handleDelete} />
  </CwTooltip>
  ```

### CwTag
* 用於顯示標籤，常用於多選輸入框中顯示已選擇的項目。
* 高度固定 20px，寬度自動適應內容。
* 背景色 #e9ebf2，圓角 5px，padding 4px 水平 / 2px 垂直。
* 文字使用 Noto Sans TC，14px，font-weight normal，單行顯示。
* 可選擇性顯示關閉按鈕（14px X 圖標），點擊可移除標籤。
* 多個標籤之間建議使用 gap-2（8px）。
* 使用場景：多選輸入框、篩選條件顯示、標籤管理。

### StatusTag
* 用於表格中顯示狀態標籤的簡單元件，不帶關閉按鈕。
* 高度固定 20px，寬度自動適應內容。
* 圓角 5px，padding 4px 水平 / 2px 垂直。
* 文字使用 Noto Sans TC，12px，font-weight 350，單行顯示。
* **顏色變體**：
  - `default`：灰色背景 #e9ebf2，黑色文字 #1c1c1c
  - `primary`：淡藍色背景 #d1ebff，藍色文字 #01579b
  - `success`：淡綠色背景 #e8f5e9，綠色文字 #568b53
  - `warning`：淡橙色背景 #fff4e5，橙色文字 #f57c00
  - `error`：淡紅色背景 #fff6f4，紅色文字 #c00000
  - `info`：淡藍色背景 #e3f2fd，藍色文字 #01579b
* **使用規則**：
  - 主要用於表格中顯示狀態、類別等固定內容
  - 透過不同顏色變體凸顯標記差異
  - 不可點擊，僅作為視覺標記
  - 不應與 CwTag（帶關閉按鈕）在同一場景混用
* **使用場景**：表格狀態欄位、訂單狀態、會員等級、審核狀態等。

### CwInput
* 單行輸入框元件，適合短字輸入。
* **尺寸與樣式**：
  - 輸入框高度：35px
  - 圓角：4px（var(--radius)）
  - 背景：白色（正常）、#e9ebf2（禁用）、#fff6f4（錯誤）
  - 邊框：#7c808c（正常）、#0078d4（focus）、#c00000（錯誤）
  - Focus 陰影：0px 0px 4px 0px #01579b
  - 文字：Noto Sans TC，14px，font-weight 350
  - Placeholder 顏色：#cdcdcd
  - 錯誤訊息：紅色文字 #c00000，12px
* **基本功能**：
  - 必須有 placeholder 提示文字
  - 視情況顯示 label 標題
  - 支援 clearable 屬性，顯示清除按鈕
  - 支援 leftIcon 和 rightIcon 顯示圖標
  - 支援 isSearch 屬性，自動在左側顯示搜尋圖標
  - 支援 textAlign 調整文字對齊方式（left/center/right）
  - 支援錯誤狀態，顯示錯誤訊息和紅色邊框
* **帶 icon 按鈕的輸入框（重要規範）**：
  - 在 input 元件中如果有使用 icon 按鈕，**不應該使用 CwRoundButton**
  - 應使用單一 icon（來自 lucide-react），並放置在 input 內的最右側
  - Icon 尺寸建議為 16px
  - Icon 顏色建議為 #7c808c（灰色）或 #01579b（主色）
  - 必須包裹在 button 元素中，具備點擊功能和 hover 效果
  - **Tooltip 使用規範**：
    - **需要 CwTooltip**：複製、另開網址
    - **不需要 CwTooltip**：清除按鈕、搜尋圖標、密碼顯示/隱藏、發送訊息
  - 常見應用：搜尋按鈕、密碼顯示/隱藏、發送訊息、複製、另開網址等
* **多選輸入框（CwInputWithTags）**：支援選擇多個項目，使用 CwTag 顯示已選擇的內容。
  - 標籤顯示在輸入框內，可以不斷增加
  - 每個標籤都有關閉按鈕，點擊可取消選取
  - 輸入框會隨標籤數量自動調整高度（最小高度 35px）
  - 輸入值後按 Enter 或觸發 onAddTag 添加標籤
  - 支援清除全部標籤功能（clearable 屬性）
  - **當已有標籤時，不顯示 placeholder**，避免視覺干擾
  - 適用場景：社工選擇、標籤管理、多人員選擇等
* **使用規則**：
  - 必須提供清楚的 placeholder 或 label 說明輸入內容

### CwSelect
* 下拉選擇元件，用於從預定義的選項列表中選擇一個或多個項目。

  - 文字：Noto Sans TC，14px，font-weight 350
  - Placeholder 顏色：#cdcdcd
  - 下拉選單：白色背景，圓角 4px，shadow，最大高度 280px

  - 項目高度：35px，padding 12px

  - 顯示「已選擇 N 項」文字提示
  - 選單保持開啟，可連續選擇多個項目

  - 找不到符合項目時顯示「找不到符合的選項」

  - 選項順序需有邏輯性，以使用者方便或能夠理解的方式排列（如：依字母或常用順序）
  - 多選的情境下，應明確顯示選取項目的數量或使用 tag 標記
  - 不可放入過多選項（建議 10 個以內），當選項過多時，應啟用 searchable 支援輸入搜尋
  - 避免使用技術文字的選項敘述，選項值與顯示文字要一致且易懂
  - 當有值時，可以點擊 clear 按鈕清除內容（需啟用 clearable）
  - 禁用選項應明確標示（disabled 屬性）
  - 錯誤狀態需顯示錯誤訊息（error 和 errorMessage 屬性）
* **主要屬性**：
  - `options`：選項列表，包含 value、label 和 disabled
  - `value`：當前選中的值（單選為 string，多選為 string[]）
  - `onChange`：值改變回調函數
  - `placeholder`：提示文字（預設：請選擇）
  - `label`：標籤文字
  - `multiple`：是否多選（預設 false）
  - `searchable`：是否可搜尋（預設 false）
  - `clearable`：是否顯示清除按鈕（預設 false）
  - `disabled`：是否禁用（預設 false）
  - `error`：是否顯示錯誤狀態（預設 false）
  - `errorMessage`：錯誤訊息文字
  - `width`：寬度（預設 100%）
* **適用場景**：
  - 類別選擇（部門、狀態、等級等）
  - 人員選擇（單選或多選）
  - 表單下拉選單
  - 篩選條件選擇
  - 任何需要從預定義選項中選擇的場景
* **綜合應用範例**：CwSelectExamples.tsx 展示了單選、多選、可搜尋、禁用選項、錯誤狀態等各種使用情境。

### CwPopup
* 彈出視窗元件，用於暫時中斷流程，提供額外資訊、確認操作或顯示重要訊息。
* **尺寸規格**：
  - `sm`：固定寬度 400px，適用於簡單確認或短訊息提示
  - `md`（預設）：最大寬度 800px，最小寬度 400px，適用於一般表單或內容展示
  - `lg`：最大寬度 976px，最小寬度 400px，適用於複雜表單或需要較寬空間的內容
  - `xl`：最大寬度 1080px，最小寬度 400px，適用於大量資訊展示，例如詳細報表或多欄位表單
  - `fit`：最大寬度為視窗寬度的 80%，最小寬度 400px，會隨視窗尺寸自動調整
* **樣式設定**：
  - 圓角 6px（var(--radius-card)）
  - 白色背景，半透明黑色遮罩（rgba(0, 0, 0, 0.5)）
  - z-index: 4000
* **結構組成**：
  - **Header**（可選）：包含標題和關閉按鈕（X）
  - **Body**：內容區域，padding 20px
  - **Footer**（可選）：包含操作按鈕
* **類型與圖標**：
  - `info`：資訊提示，顯示 Info 圖標
  - `success`：成功提示，顯示 CheckCircle 圖標
  - `warning`：警告提示，顯示 AlertTriangle 圖標
  - `error`：錯誤提示，顯示 XCircle 圖標
  - `question`：問題提示，顯示 HelpCircle 圖標
  - 無類型：不顯示圖標
* **定位方式**：
  - 所有 Popup 統一距離頂部 70px，水平居中
  - 確保在不同螢幕尺寸下保持一致的視覺位置
* **遮罩行為**：
  - `closableByMask={true}`（預設）：可點擊遮罩關閉，適合一般資訊展示
  - `closableByMask={false}`：不可點擊遮罩關閉，適合重要確認操作
  - 按 ESC 鍵也可關閉（當 closableByMask 為 true 時）
* **可滾動內容**：
  - 當內容過多時，設定 `scrollGradient={true}` 啟用滾動
  - 設定 `maxHeight` 控制最大高度（預設 646px）
  - 滾動時顯示頂部和底部的漸變提示
* **按鈕配置**：
  - 使用 `buttons` 屬性配置 Footer 按鈕
  - 按鈕順序：取消(outline) > outline > 實心 > 刪除(實心)
  - 每個按鈕包含：label、variant、appearance、onClick
* **使用規則**：
  - 必須有明確的關閉機制（Header 關閉按鈕或 Footer 按鈕）
  - 可選擇性隱藏 Header 或 Footer，但不可同時隱藏兩者
  - 禁止同頁同時開啟多個 Popup
  - 使用多步驟時，透過關閉與開啟的機制交替使用
  - 根據內容量選擇適當的尺寸規格
  - 避免在 Popup 中放置過多資訊或複雜操作流程
  - 按鈕文字應簡短並能說明意圖
  - 重要確認操作必須設定 `closableByMask={false}`
* **適用場景**：
  - 確認刪除、提交、離開等重要操作
  - 顯���操作結果（成功、失敗、警告）
  - 提供額外資訊或說明
  - 需要使用者回答問題或做出選擇
  - 顯示使用條款、隱私政策等長文內容（配合滾動功能）

### CwTable
* 通用表格元件，使用真正的 `<table>` 元素配合 `table-layout: fixed`，確保每行同一個欄位的寬度相同並與表頭完美對齊。
* 支援自定義渲染、空狀態等功能。**不顯示排序圖標**，需要排序功能請使用 CwSortTable。
* **欄位寬度規則**：
  - 欄位可以填滿整個畫面，支援固定寬度和自動填滿的混合使用
  - 設定 `width` 屬性（如 "120px"）的欄位會使用固定寬度
  - 未設定 `width` 的欄位會自動填滿剩餘空間（平均分配）
  - 當內容過多或多個欄位被固定寬度時，表格會自動啟用左右捲動
* **欄位定義（CwTableColumn）**：
  - `key`：欄位唯一識別
  - `title`：欄位標題
  - `width`：欄位寬度（使用像素值，例如 "120px" 或 "60px"；不設定則自動填滿）
  - `render`：自定義渲染函數，可用於顯示 StatusTag、操作按鈕或其他自定義內容
  - `align`：欄位對齊方式（'left' | 'center' | 'right'）
* **狀態標籤**：可使用 StatusTag 元件顯示狀態等固定內容項目，透過不同顏色變體快速辨認資料狀態。
* **橫向捲動**：當欄位過多時，自動啟用捲軸使資料集可橫向捲動閱讀，避免資料集過度擁擠或內容截斷。
* **空狀態**：
  - 當 dataSource 為空時，自動顯示 CwEmptyState 元件
  - 包含圖像（100px × 100px）和文字提示
  - 可通過 `emptyText` 屬性自訂提示文字（預設：沒有資料）
  - 空狀態會垂直置中顯示，並有適當的上下間距
* **操作欄位**：通常在最後一個欄位放置操作按鈕（CwRoundButton），並設定固定 width 和 align 為 'center'。
* **文字處理**：
  - 所有欄位都使用 break-words 換行顯示長文字，不會出現截斷符號（...）
  - 內容過多時會自動折行，確保所有內容完整顯示
  - 英文單詞過長時會自動斷行
* **文字對齊**：除特殊情況外，表格中的文字與內容以「靠左對齊」為主，保持閱讀的一致性。
* **對齊保證**：使用 `<colgroup>` 和 `<col>` 元素確保每行同一個欄位的寬度完全相同，並與表頭完美對齊。
* 視情況可搭配 CwPagination 元件實現分頁功能。
* **使用場景**：不需要排序功能的資料展示表格。需要排序功能時請使用 CwSortTable。

### CwSortTable
* 帶排序功能的表格元件，在 CwTable 基礎上增強了內建客戶端排序功能。
* 使用者可點選欄位標頭旁的排序圖標，進行升序或降序排列。
* **���序圖標狀態**：
  - 預設（未排序）：雙向箭頭（灰色 #7c808c）
  - 升序：向��箭頭（藍色 #01579b）
  - 降序：向下箭頭（藍色 #01579b）
  - 取消排序：回到預設狀態
* **排序邏輯**：
  - 第一次點擊：升序排列
  - 第二次點擊：降序排列
  - 第三次點擊：取消排序，恢復原始順序
  - 內建支援數字、字串（使用 zh-TW 本地化比較）、null/undefined 處理
* **固定顯示內容**：可用 StatusTag 元件顯示狀態等固定內容項目，透過不同顏色變體方便使用者快速辨認資料狀態。
* **橫向捲動**：當欄位過多時，自動啟用捲軸使資料集可橫向捲動閱讀，避免資料集過度擁擠或內容截斷。
* **文字對齊**：除特殊情況外，表格中的文字與內容以「靠左對齊」為主，保持閱讀的一致性。
* **使用規則**：
  - 將需要排序的欄位設定 `sortable: true`
  - 不需要排序的欄位（如操作按鈕）不設定 sortable
  - 可通過 `onSort` 回調自定義排序邏輯（例如後端排序）
  - 表格內容不可出現截斷符號（...），內容過多時自動折行
* **適用場景**：會員列表、訂單管理、資料報表等需要排序功能的表格頁面。
* **綜合應用範例**：訂單查詢頁面（OrderManagement.tsx）展示了完整的應用情境，包含搜尋表單、StatusTag 狀態標籤、日期範圍選擇、排序功能和分頁控制。

### CwTableSet
* 整合標題、搜尋、表格（CwSortTable）和分頁的組合元件，提供完整的資料展示方案。
* **組成元素**：
  - **標題列**：顯示表格標題，右側可選擇性顯示搜尋框
  - **資料表格**：使用 CwSortTable 展示資料，支援排序功能
  - **分頁控制**：使用 CwPagination 控制分頁
* **主要特色**：
  - 整合式設計：一次配置所有表格展示相關功能
  - 搜尋功能：支援即時搜尋過濾（需在父元件處理過濾邏輯）
  - 排序功能：繼承 CwSortTable 的多欄位排序能力
  - 分頁控制：自動計算總頁數和資料範圍
  - 彈性配置：可選擇性顯示搜尋框和分頁
* **使用規則**：
  - 搜尋功能需要在父元件處理過濾邏輯
  - 分頁資料需要在父元件進行切片處理
  - totalItems 應傳入過濾後的總筆數
  - 搜尋或改變每頁筆數時，建議將 currentPage 重置為 1
  - 支援所有 CwSortTable 的欄位配置選項
* **適用場景**：
  - 需要完整的資料展示功能（標題、搜尋、排序、分頁）
  - 快速建立標準化的資料管理頁面
  - 需要一致的表格展示風格
* **綜合應用範例**：產品管理頁面（ProductManagement.tsx）展示了完整的應用情境，使用 CwTableSet 實現產品列表的搜尋、排序、分頁和狀態標籤展示。

### CwToast
* 短暫出現的訊息通知元件，用於提示操作結果或重要訊息。
* **位置與樣式**：
  - 固定在頁面頂部 70px，水平居中
  - 最大寬度 460px，最小高度 40px
  - 圓角 4px，白色文字配合不同類型的背景色
  - z-index: 2000
* **Toast 類型與顏色**：
  - `info`：資訊提示，藍色背景 (#01579b)，顯示 Info 圖標（i）
  - `success`：成功提示，綠色背景 (#7cb342)，顯示勾勾圖標
  - `warning`：警告提示，橙色背景 (#e79f2b)，顯示警告圖標（!）
  - `error`：錯誤提示，紅色背景 (#c00000)，顯示錯誤圖標（X）
  - `question`：問題提示，淺綠色背景 (#7ac29a)，顯示問號圖標（?）
* **功能特性**：
  - 預設 3 秒後自動關閉（可通過 duration 屬性自訂）
  - 可選擇性顯示關閉按鈕（closable 屬性）
  - 帶關閉按鈕的 Toast 可設定 duration 為 0，需手動關閉
  - 訊息文字使用 Noto Sans TC Medium，14px
* **使用規則**：
  - 必須使用簡潔、單句的訊息通知
  - 避免一次出現多則 Toast，同時只應顯示一個
  - 避免過長的文字內容，減少閱讀時間
  - 若訊息較長不可避免，需加長適切的顯示時間或使用可關閉按鈕
  - 需使用明確的顏色/圖示區分成功、錯誤、警告等類型
  - 禁止在 Toast 中放置操作按鈕或複雜內容
* **適用場景**：
  - 操作成功提示（儲存成功、刪除成功等）
  - 操作失敗提示（輸入錯誤、系統錯誤等）
  - 警告提示（條件不足、格式錯誤等）
  - 資訊提示（一般性通知）
  - 問題提示（需要確認或釐清的情況）

### CwTooltip
* 提示元件，用於滑鼠 hover 時顯示補充資訊。通常與 CwRoundButton 搭配使用。
* **位置與樣式**：
  - 背景色：rgba(28, 28, 28, 0.8)，半透明黑色
  - 圓角：4px
  - 內距：px-[5px] py-[2px]
  - 最大寬度：150px，超過會自動換行
  - 文字：Noto Sans TC，14px，font-weight 350，白色
  - 帶有三角形箭頭指向觸發元素
  - z-index: 3000
* **位置選項**：
  - `top`：出現在觸發元素上方（預設）
  - `bottom`：出現在觸發元素下方
  - `left`：出現在觸發元素左側
  - `right`：出現在觸發元素右側
* **智能定位**：
  - 自動計算位置確保 Tooltip 不超出視窗邊界
  - 根據觸發元素位置自動調整，保持 10px 間距
  - 箭頭會根據位置自動調整方向
* **功能特性**：
  - 滑鼠移入觸發元素時顯示
  - 滑鼠移出時隱藏
  - 可通過 `disabled` 屬性禁用
  - 不可點擊，pointer-events-none
* **使用規則**：
  - 必須使用簡潔、單句的文字說明
  - 避��放置段落型文字或複雜內容
  - 文字建議不超過一行（約 15 個中文字）
  - 需明確標示觸發區域（按鈕、圖標、文字）
  - 提示內容不應包含使用者完成任務所必需的關鍵資訊或操作
  - 禁止在 Tooltip 中放置操作按鈕或互動元素
  - 避免與其他元件內容重疊（系統會自動調整）
  - 確保在各種情形下文字可被正確閱讀
* **適用場景**：
  - 補充說明按鈕或圖標的功能（如：檢視、刪除、下載）
  - 提供額外的上下文資訊
  - 解釋縮寫或專業術語
  - 說明禁用或不可用的原因
* **不適用場景**：
  - 完成任務所必需的關鍵資訊（應使用 CwPopup）
  - 包含操作按鈕或表��（應使用 CwPopup 或 CwDrawer）
  - 段落型長文內容（應使用 CwPopup）
  - 觸控裝置（無 hover 狀態，應考慮其他方案）
* **常見組合**：通常與 CwRoundButton 搭配使用，為表格操作按鈕提供功能說明。

### CwDrawer
* 從畫面右側滑出的抽屜元件，用於呈現補充內容或可編輯內容。
* **位置與樣式**：
  - 從右側滑入，固定在頁面右側
  - 預設寬度：400px
  - 可調整範圍：300px - 800px
  - 高度：100%，隨視窗高度自動延伸
  - 圓角：左側 16px（var(--radius-drawer)），右側 0
  - 半透明黑色遮罩：rgba(0, 0, 0, 0.5)
  - z-index: 1000
* **結構組成**：
  - **頂部操作區**：包含關閉按鈕、可選的上下切換按鈕
  - **內容區域**：可滾動的主要內容區，padding 24px
  - **可調整大小**：拖動左側邊緣可調整寬度
* **功能按鈕**：
  - 關閉按鈕（X 圖標）：永遠顯示在最左側
  - 上一筆按鈕（↑）：可選，用於逐筆切換時
  - 下一筆按鈕（↓）：可選，用於逐筆切換時
  - 所有按鈕 hover 時顯示背景色變化
* **互動行為**：
  - 開啟時防止背景頁面滾動
  - 點擊遮罩預設可關閉（可設定 closableByMask={false} 禁用）
  - 拖動左側邊緣可調整寬度（可設定 resizable={false} 禁用）
  - 換頁或進行額外操作時應自動關閉
* **使用規則**：
  - 必須具備明確的關閉按鈕
  - 避免放入過長或多層次內容，導致使用者迷失方向
  - 用於補充性、次要但需要互動的內容顯示
  - 避免在同一個頁面中出現多個抽屜元件
  - 禁止多個 Drawer 同時顯示
  - 重要編輯操作建議設定 closableByMask={false}
  - 標題應簡潔明確，說明抽屜內容用途
* **適用場景**：
  - 表單編輯（會員資料、訂單資訊等）
  - 資料檢視（詳細資料展示）
  - 逐筆瀏覽（配合上下按鈕切換）
  - 篩選器或設定面板
  - 任何需要保持主頁面可見的補充內容
* **不適用場景**：
  - 複雜的多步驟流程（應使用多個頁面或 Popup）
  - 需要使用者完全專注的重要操作（應使用全螢幕頁面）
  - 過長的內容或多層嵌套結構
  - 同時需要顯示多個不同資訊的情況
* **與其他元件的區別**：
  - **vs CwPopup**：Drawer 從側邊滑出，適合較多內容；Popup 居中彈出，適合關鍵操作確認
  - **vs CwTooltip**：Drawer 可互動、內容豐富；Tooltip 僅用於簡短提示
  - **vs 全螢幕頁面**：Drawer 保持主頁面可見，適合補充性內容；新頁面適合完整的工作流程
