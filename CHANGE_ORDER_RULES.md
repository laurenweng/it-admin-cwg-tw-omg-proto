# 異動單規則

## 退件補寄單（RSD）

### 業務定義

**退件**
收到瑕疵品後可以執行退件，讓發行將瑕疵書收回倉庫。

**補寄**
客戶反應未收到產品，需執行補寄。一般商品及雜誌均可執行，數位產品不適用。

### 操作類型

退件補寄單分三種情境，popup 內以 radio 選擇：

| 操作類型 | 代碼 |
|----------|------|
| 純退件 | `return-only` |
| 純補寄 | `resend-only` |
| 退件＋補寄 | `both` |

開啟舊記錄時推導：有 `returnReason` + 有 `resendReason` → both；僅有 `resendReason` → resend-only；其餘 → return-only。新建單預設 `both`。

### 按鈕規則

| 操作類型 | 清除 | 取消 | 存檔 | 核單 |
|----------|------|------|------|------|
| 純退件 | ✓ | ✓ | ✓ | ✗ |
| 純補寄 | ✓ | ✓ | ✓ | ✗ |
| 退件＋補寄 | ✓ | ✓ | ✓ | ✓ |

### 暫停處理 / 暫止原因 規則

純補寄（`resend-only`）時：
- 暫停處理：強制顯示「是」（disabled，不可更改）
- 暫止原因：必填，CwSelect 啟用

其他類型：
- 暫停處理：顯示既有值
- 暫止原因：CwSelect disabled

### 暫止原因選項

- `01_單純新增退件單`
- `02_補寄且暫不出貨`
- `03_中獎電子發票無地址可寄`

### 存檔警語

存檔前須跳出確認 popup，告知：「退件資訊儲存後將直接觸發入庫作業，請確認資訊正確無誤後再執行存檔。」

### 欄位結構

#### 固定欄位（popup）

**左欄**
- 退件補寄單號（disabled）
- 退件補寄單日期（disabled）
- 退件補寄單狀態（disabled）
- 原訂單號碼（白底 + … 查詢按鈕）
- 訂單客戶編號（disabled）
- 退件原因（edit/create → CwSelect；view → 灰底 span）
- 實際退件單日期（disabled）
- 暫停處理（dropdown）
- 暫止原因
- 出貨注意事項

**右欄**
- 核單日期（disabled）
- 訂單類型（白底）
- 訂單客戶名稱（disabled）
- 聯絡方式（disabled）
- 出貨方式（白底）
- 出貨客戶名稱（disabled）
- 出貨地址（disabled）
- 出貨客戶編號（disabled）
- 出貨收件人（disabled）
- 補寄原因（edit/create → CwSelect；view → 白底 span）

**備註欄**（右側獨立 textarea）

#### 補寄明細 table 欄位

訂單料號 / 訂單產品 / 數量 / 出貨倉 / 發行出貨 / 出貨日期 / 出貨客戶名稱 / 出貨收件人 / 出貨地址 / 下發註記 / 處理狀態 / 執行狀態

### 新增流程（從新增異動單 popup）

1. 訂單號碼（預帶入）
2. 選擇異動類型
3. 選擇出貨明細（RSD 時顯示出貨資訊 tab 的列表；其他類型顯示訂單明細列表）

切換異動類型時，已選明細清空。
